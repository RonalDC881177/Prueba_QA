# Prueba_QA

## Analisis y planteamiento.

Se Diseña un plan de pruebas enfocado en validar los flujos de la aplicación de gestión de contactos, garantizando estabilidad en autenticación, CRUD de contactos e integraciones entre UI y API.

## estrategia de pruebas por priorizacion:

Se define las pruebas a realizar separando en 3 nieveles segun el tipo de validacion a realizar de lasiguiente manera:

1. Pruebas API: se deja como prueba inicial ya que son mas rapidas y permiten una mayor cobertura segun lo solicitado.

2. UI: Se automatizan los flujos criticos validando la correcta visualizacion y funcionamiento.

3. Integracion: Se valida la integracion entre UI y API validando quelo creado en una se visualiza en la otra.

## Plan de pruebas:

Se establece el plan de pruebas siguiendo la priorizacion y se Incluyen tanto casos positivos como negativos para validar comportamiento esperado y manejo de errores.
